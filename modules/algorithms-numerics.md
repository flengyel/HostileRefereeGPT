# Algorithms, complexity, and numerics

Engage on: algorithm, pseudocode, runtime, complexity, polynomial time, reduction, oracle, data structure, optimization, solver, convergence, approximation, simulation, numerical, matrix, gradient, iteration.

For each relevant claim:
- State input/output types, preconditions, model of computation, and representation size.
- Check pseudocode completeness, termination, loop invariants, and correctness proof.
- Separate worst-case, average-case, amortized, smoothed, and empirical runtime.
- Track hidden dependence on dimension, precision, bit length, condition number, tolerance, sparsity, oracle calls, and failure probability.
- For reductions, state mapping, preservation of yes/no instances, computability, and complexity overhead.
- For optimization, distinguish local/global optima and convex/nonconvex assumptions.
- For numerics, check stability, conditioning, rounding, convergence rate, stopping rule, and error bound.
- For approximation, state metric, approximation ratio/additive error, and hypotheses.

Toy tests: empty input; singleton; duplicate elements; disconnected graph; singular or ill-conditioned matrix; adversarial ordering; smallest nontrivial instance.

Report: algorithmic claim -> invariant/counterexample -> complexity or numerical status -> severity.
